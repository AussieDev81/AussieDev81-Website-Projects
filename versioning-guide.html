How it works

This tools functionality was adopted (and simplified) from the Semantic Versioning 2.0.0 specification.

Numeric suffixes to the pre-release name (E.g. 1.0.0-alpha-2) are not yet supported, perhaps in a future version ;-), however pre-release names are automatically generated (in order) following the Greek Alphabet

For existing pre-release names, the system reads the first character of the given pre-release, and automatically defaults to the corresponding Greek Alphabet name, for example 1.0.0-apples becomes 1.0.0-alpha.
Additionally, still using the example of 1.0.0-apples, if the user declares that the version is potentially instable, the pre-release will increment to the next Greek Alphabet letter resulting in the version number 1.0.0-beta
